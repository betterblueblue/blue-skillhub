## 强制规则（违反即 FAIL）

> 仅当"校验手段"能真正落地拦截时才标"强制"。
> 能 grep 的写 grep 模式；需语义判断的写"人工确认"；不确定的先标"建议"。

| 规则 | 校验手段 | 说明 |
|------|---------|------|
| <!-- 例：禁用 `System.out`，日志走 `BizLogger` --> | <!-- grep:`System\.out` --> | <!-- 日志统一 --> |
| <!-- 例：禁止组件内裸 `axios`，统一走 `src/api/` --> | <!-- grep-exclude:`axios`:`src/api/` --> | <!-- API 层统一 --> |
| <!-- 例：对外接口必须返回 `Result<T>` --> | <!-- 人工确认返回类型 --> | <!-- 自研封装 --> |

**校验手段写法参考**：

| 写法 | 含义 | impact_validate.py V8 行为 |
|------|------|---------------------------|
| `grep:<正则>` | 在变更文件中搜索该正则 | 命中"禁用/禁止"规则时 FAIL |
| `grep-exclude:<正则>:<目录>` | 搜索正则但排除指定目录 | 排除后命中"禁用/禁止"规则时 FAIL |
| `人工确认` | 需要语义判断，grep 做不到 | WARN，列入"需人工复核"清单 |
| `code-graph + 人工` | 有 code-graph MCP 时缩小候选，无则人工 | WARN，列入"需人工复核"清单 |
