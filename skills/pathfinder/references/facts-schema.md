# Pathfinder Facts Schema

`change-impact/_project-map/facts/` contains machine-readable facts generated by Pathfinder scripts. These files are writable because they are Pathfinder-managed outputs; they are not project source files.

## Shared Metadata

Every facts JSON file must include:

| Field | Type | Meaning |
|---|---|---|
| `schema_version` | number | Current value: `1` |
| `generator` | string | Script name, such as `pf_scan.py` or `pf_git.py` |
| `source_path` | string | Project root scanned by the script |
| `observed_at` | string | ISO timestamp when the facts were observed |

`pf_validate.py` checks these fields as part of the `_project-map.md` Script Gate.

## `git.json`

Required fields:

| Field | Type | Meaning |
|---|---|---|
| `is_git_repo` | boolean | Whether the target path is inside a Git repository |
| `is_independent_repo` | boolean | Whether `.git` belongs directly to the target project root |
| `toplevel` | string/null | Git toplevel path, or null for non-Git |
| `head_short` | string/null | Current short HEAD for independent repos; null otherwise |
| `head_full` | string/null | Current full HEAD for independent repos; null otherwise |
| `branch` | string/null | Current branch for independent repos; null otherwise |
| `hotspots` | object[] | Recently changed files |
| `recent_commit_modules` | string[] | Top-level modules inferred from hotspots |

For non-Git or non-independent project directories, do not copy parent-repo HEAD into `head_short`; write `null` instead.
