# impact-pro 第十一轮验收总结（定级责任与时机修正）

- 测试日期：2026-06-07
- 测试方式：流程规则审查 + 文档同步
- 目标：明确 light/full 不应在用户模糊需求后立即定档，而应在证据发现和苏格拉底式澄清后正式判定
- 结论：**定级流程已改为 Agent 预判 + 苏格拉底澄清 + Agent 正式建议 + 用户复核确认**

## 修正前问题

原流程是：

```text
Phase 2 技术栈检测/上下文发现
→ Phase 2.5 定级确认
→ Phase 3 苏格拉底式探索
```

这个顺序容易导致需求尚未澄清就提前决定 light/full。

## 修正后流程

```text
Phase 2 技术栈检测/上下文发现
→ Phase 2.5 初步风险预判（不最终定档）
→ Phase 3 苏格拉底式探索
→ Phase 3.5 正式定级 + 用户确认
```

## 责任边界

- Agent 负责初步风险预判和正式档位建议。
- Agent 必须列出允许 light 的证据、触发 full 的证据、未确认项。
- 用户负责复核确认、补充业务决策或要求升档。
- 用户可以要求简化 full 输出，但不能把已触发 full 的高风险变更降为跳过安全闸的 light。

## 本轮改动

- `SKILL.md`：新增 Phase 2.5 初步风险预判，新增 Phase 3.5 正式定级。
- `README.md`：同步典型流程和定级说明。
- `VALIDATION.md`：验收标准明确“一句模糊需求后直接定档”是失败信号。

## 当前结论

```text
light/full 的正式判定时机 = 只读证据发现 + 苏格拉底式澄清之后，文档输出之前。
```
