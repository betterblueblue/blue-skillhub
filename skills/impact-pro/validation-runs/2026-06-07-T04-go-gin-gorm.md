### 风险追问

应追问：

1. `last_login_at` 是否只在登录成功时更新，登录失败是否不更新？
2. 时间使用 UTC 还是本地时区？
3. GORM 字段是 `time.Time` 还是 `*time.Time`，历史用户默认值如何处理？
4. 是否通过 serializer 暴露给 `/api/user` 或 profile 接口？
5. AutoMigrate 是否允许生产环境自动变更 schema，是否需要显式迁移方案？

### 判档

应判定 full。

理由：

- 涉及 GORM model 字段。
- 涉及登录业务逻辑和认证流程。
- 涉及 API serializer 输出。
- 涉及测试和时间边界。
- AutoMigrate 不等同可靠回滚，需要明确生产迁移策略。
