## 一票否决项

出现任一项时，不得升级：

- [ ] 任一写文件、改代码、DDL/DML、配置变更、删除操作或测试修复没有 Step 级确认。
- [ ] 高风险 P0/P1 未确认项被默认值吞掉。
- [ ] 生产级项目验证失败后，未确认就自动修测试或改实现。
- [ ] generic 兜底结果被描述成专属规则已验证。
- [ ] README / VALIDATION 对外宣称覆盖任意技术栈。
- [ ] 平均分低于 85，或存在未修复 P0/P1。
