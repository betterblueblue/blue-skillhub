# V10 盲测设计 — v4.1 多轮触发 + 链路追踪回流验证

> 评测日期：2026-06-25
> 测试模型：Composer 2.5（唯一）
> Skill 版本：v4.1（v4.0 全部改进 + 多轮触发条件 + 链路追踪发现回流 Phase 3）
> 评测范围：仅 B3'（prisma-express-ts 邮箱验证强制检查），单 case 验证

---

## 一、V10 核心目标

### 1.1 要回答的问题

V9 人工交互盲测中，agent 在 B3' 链路追踪发现了"send-verification-email 需改为公开接口"的方案，但未追问改为公开接口后的副作用风险（如"重发验证邮件可被无限调用"）。agent 只问了 1 轮 3 题就停止了。v4.1 新增了多轮触发条件和链路追踪回流自检。

> **v4.1 是否让 agent 在第 1 轮 3 题之后，基于链路追踪发现的副作用风险继续追问第 2 轮？**

### 1.2 与 V9 的区别

| 维度 | V9 | V10 |
|------|-----|-----|
| Skill 版本 | v4.0 | **v4.1**（新增多轮触发 + 链路追踪回流） |
| Case 范围 | B1'/B2'/B3' 三个 | **仅 B3'**（验证 v4.1 的目标 case） |
| 交互模式 | 人工交互 | 人工交互（同 V9） |
| noskill 对照 | 复用 V8 | 复用 V9 |
| 评审重点 | 人工交互有效性 + v4.0 特性 | **v4.1 多轮触发 + 链路追踪回流** |

### 1.3 为什么选 B3'

V9 B3' 是暴露"链路追踪发现但不追问"问题的具体 case：
- 链路追踪 §5"数据流路径"行发现：send-verification-email 需改为公开接口
- 但改为公开接口后可被无限调用（P1 级副作用风险）
- V9 agent 只问了 1 轮 3 题就停止，未追问此风险
- v4.1 的链路追踪回流自检（步骤 2）要求扫描 §5"发现的二级影响"列，标记需要用户决策的副作用风险点

### 1.4 与 V9 的相同点

- 用户原话不变："注册的时候不是有发验证邮件吗，但是不验证好像也能用，这样不行吧"
- 测试项目不变（prisma-express-ts 独立副本）
- 使用 impact-pro skill
- 人工交互模式（agent 暂停等待用户回答）

## 二、人工交互设计

### 2.1 交互流程

与 V9 一致：

```
Agent (skill v4.1)               Human (真实开发者)
    |                                  |
    |  Phase 1-2: 读取项目、分析上下文    |
    |  （含链路追踪 §5 二级影响）         |
    |                                  |
    |  Phase 3: 第 1 轮澄清问题         |
    |--------------------------------->|
    |                                  |  根据自身判断回答
    |<---------------------------------|
    |                                  |
    |  v4.1 自检：扫描 §5 二级影响       |
    |  发现未覆盖的 P0/P1 副作用风险？    |
    |  ↓ 是 → 进入第 2 轮               |
    |  Phase 3: 第 2 轮澄清问题         |
    |--------------------------------->|
    |                                  |  回答
    |<---------------------------------|
    |                                  |
    |  Phase 4: 基于用户回答产出文档      |
    |  Phase 5: 执行前检查              |
    |                                  |
```

### 2.2 核心约束

1. 人工只回答 Phase 3 的澄清问题，不参与其他环节
2. 人工以真实开发者身份回答
3. 对话记录就是交互记录
4. 第 1 轮回答与 V9 保持一致（确保可比性）

### 2.3 第 1 轮回答预案

为保持与 V9 的可比性，第 1 轮回答与 V9 一致：

| V9 问题 | V9 回答 | V10 第 1 轮回答 |
|---------|---------|----------------|
| Q1: 未验证用户在哪拦截？ | A) 登录时直接拒绝，不发 token | 同 V9 |
| Q2: 存量未验证用户怎么处理？ | A) 同样适用，不豁免 | 同 V9 |
| Q3: 拦截范围？哪些接口可用？ | A) 仅 verify-email、重发验证邮件、logout | 同 V9 |

第 2 轮（如果触发）的回答由开发者根据问题内容自然回答。
