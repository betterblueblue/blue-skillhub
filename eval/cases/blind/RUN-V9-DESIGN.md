# V9 盲测设计 — 人工交互回归验证 + v4.0 改进首发验证

> 评测日期：2026-06-25
> 测试模型：Composer 2.5（唯一）
> 评测目标：
> 1. 验证人工交互场景下 skill 的澄清流程是否产生比"自问自答"更好的分析质量
> 2. 验证 v4.0 的 2 项改进（配置依赖链路追踪 + light 配置化提示）是否生效

## 一、V9 核心目标

### 1.1 要回答的问题

V8 验证了 v3.9 改进在"agent 自问自答"模式下全面生效。但 V8 的 skill 组有一个不真实的设定：agent 自己提出澄清问题后自己回答，假设质量取决于 agent 自身。V9 要回答：

> **当澄清问题由真实开发者回答时（模拟生产场景），skill 的分析质量是否优于"自问自答"模式？v4.0 的 2 项改进是否生效？**

### 1.2 与 V8 的区别

| 维度 | V8 | V9 |
|------|-----|-----|
| Skill 组澄清方式 | agent 自问自答，标注 `[假设]` | **人工回答**，agent 暂停等待用户输入 |
| Skill 版本 | v3.9 | **v4.0**（新增配置依赖链路追踪 + light 配置化提示） |
| 人工参与 | 无 | **有**（skill 组 Phase 3 澄清环节） |
| 评审重点 | v3.9 特性存在性 + V7 遗漏项 | **人工交互有效性 + v4.0 特性 + V8 对比** |
| noskill 组 | 与 V7 一致 | **与 V8 一致**（直接复用 V8 C1 产出） |

### 1.3 与 V8 的相同点

- 3 个 Case 不变（B1'/B2'/B3'），prompt 不变
- 测试项目不变（ruoyi-vue + prisma-express-ts）
- Skill 分配不变（B1→impact，B2/B3→impact-pro）
- 评分维度不变（D1-D7 共 100 分 + 安全门禁）

### 1.4 为什么人工交互是关键验证点

V8 的"自问自答"模式存在两个问题：
1. **假设过于完美**：agent 自己问自己答，假设可能比真实用户回答更精确
2. **无法验证澄清流程本身**：agent 可能因为"知道答案"而跳过某些该问的问题

V9 的人工交互模式可以验证：
- skill 的 Phase 3 澄清问题是否覆盖了关键模糊点
- 真实开发者的回答是否触发更好的分析方向
- 人工确认的需求（vs `[假设]`）是否产出更可执行的文档

## 二、人工交互设计

### 2.1 交互流程

```
Agent (skill)                    Human (真实开发者)
    |                                  |
    |  Phase 1-2: 读取项目、分析上下文    |
    |                                  |
    |  Phase 3: 提出澄清问题            |
    |--------------------------------->|
    |                                  |  根据自身判断回答
    |<---------------------------------|
    |                                  |
    |  Phase 4: 基于用户回答产出文档      |
    |  (010 §2.2 中标注"用户确认"        |
    |   而非 [假设])                     |
    |                                  |
    |  Phase 5: 执行前检查              |
    |                                  |
```

### 2.2 核心约束

1. **人工只回答 Phase 3 的澄清问题**，不参与其他环节
2. **人工以真实开发者身份回答**——不需要角色扮演，不需要看预设答案
3. **对话记录就是交互记录**——不需要额外维护 transcript 文件，评审时直接看对话上下文
4. **如果 agent 没有提问就直接产出文档**，记录为"未触发澄清流程"（扣 D5 分）

### 6.2 D5 评分变化（V8→V9）

V8 的 D5 评估"假设质量"——agent 自问自答，假设是否合理。
V9 的 D5 评估"需求澄清质量"——agent 的澄清流程是否有效：

- **澄清问题覆盖度**（5 分）：agent 提出的澄清问题是否覆盖了需求中的所有关键模糊点？
- **用户回答的落用**（5 分）：agent 是否正确理解并落用了用户的回答？有没有歪曲或遗漏？
- **未确认项处理**（5 分）：当用户说"你看着办"或回答模糊时，agent 是否合理地做了 `[假设]` 并标注？
