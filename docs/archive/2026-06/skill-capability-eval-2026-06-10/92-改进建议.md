## 关键观察

### 两个 skill 的"健康度"（REAL 跑分）

- **impact**：成熟度高，多模块 / 权限 / 导出 / Vue 全栈**单 skill 完成**。R3 负向 case 完美守门。
- **impact-pro**：结构化更好，profile 引导清晰；monorepo 双 profile 加载流畅；F3 跨端 case 表现优秀。
- **共同短板**：
  - 协议默认假设交互式用户（subagent 上下文需要无人值守 fallback）
  - 模板缺"未确认项"强制段 / "Out of Scope"段
  - 无 DB MCP 时的降级路径文档化不完整
  - Pydantic / SQLAlchemy 分离（已有，但 F1 还能更深）

### 真实 subagent 跑分 vs 人审 mock 跑分

- **真实 subagent 跑分**比**人审 mock 跑分**更接近生产场景
- 真实跑分**证伪了 2 个原 P1**（EasyExcel、i18n）
- 真实跑分**发现了 2 个原 mock 未发现的问题**（F2 已有实现、R3 路径错位）
- 真实跑分**比人审浅**（alembic head 未读、admin 角色需手动 grep）
- **建议**：生产 subagent 跑分作为**自动 gate**；人审作为**深度 gate**

### 自动化程度

- subagent 跑分比人审稳定（人审有时主观），但**深度比人审略浅**
- 行为分 7-10 分稳定（7/9 case 满分 10）
- 通过率 100% 表明两个 skill 都能在受监督场景下稳定使用
- **9 case 平均 91.8 / 100** 远超 80 门槛——skill 投入生产有强证据
