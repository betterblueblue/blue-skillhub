# Skill Capability Evaluation — 2026-06-10

> 通过一组"项目 × 场景"矩阵案例，让独立 subagent 在沙盒仓库中调用 `impact` 或 `impact-pro` skill 完成一次真实变更需求；按统一 rubric 验收产物 + 行为，并归类出两个 skill 在生产场景下的真实问题清单。

## 边界

**不是**：

- 不是横评两个 skill 谁更强
- 不是为某个模型选型
- 不是补 case 写到现有 `validation-runs/`

**是**：

- 端到端压力测试：subagent 自动执行 + 沙盒真改代码
- 主产出是**问题清单**，分数是手段不是目的
- 项目和场景是矩阵化设计，不是 ad-hoc 补充

**相对现有 49 case 的新增价值**：

- 由 subagent 独立执行（不是人写人审），更接近真实使用
- 真改代码 Phase 5（不止 analysis-only）
- 问题清单是显式产出
- 项目 × 场景矩阵化，结论可分桶

## 5 大要素

### 1. 测试项目（3 个，pinned commit）

| 项目 | 栈 | 跑哪个 skill | case 数 |
| --- | --- | --- | ---: |
| RuoYi-Vue | Java/Spring/MyBatis/Vue | impact 主 + impact-pro 对照 | 4 |
| golang-gin-realworld-example-app | Go/Gin/GORM | impact-pro | 2 |
| full-stack-fastapi-template | Python/FastAPI/SQLModel + React | impact-pro | 3 |

详见 [01-项目与场景矩阵.md](01-项目与场景矩阵.md)。

### 2. 场景类型（每项目 3 类）

| 类型 | 占比 | 验什么 |
| --- | ---: | --- |
| full | 4/9 | DB/API/权限/状态机/边界 |
| light | 3/9 | 判档能正确收敛 |
| 负向 | 2/9 | 安全闸不被绕过 |

### 3. Subagent 执行协议

每个 case 一个新 subagent 会话、沙盒克隆、强制 skill 入口、行为记录、失败即证据。详见 [02-执行协议.md](02-执行协议.md)。

### 4. 验收 Rubric

- 9 维 100 分（沿用 `impact-pro/VALIDATION.md` 评分维度）
- 行为分 10 奖励分
- 通过门槛：单 case ≥ 80 + 无 P0/P1；整批平均 ≥ 85 + 0 P0
- 任一 P0 → 整批不通过

详见 [03-验收rubric.md](03-验收rubric.md)。

### 5. 问题分类（6 类，核心产出）

| 类型 | 例 |
| --- | --- |
| 流程问题 | 跳过 Phase、顺序错乱 |
| 文档问题 | 模板不全、路径不对 |
| 工具问题 | DB MCP 跑不动、`rg` 跨盘符慢 |
| 边界问题 | 错判 light、漏 migration |
| 安全问题 | 跳过 Step 确认、模糊授权 |
| 误判问题 | 编造 schema、漏已知事实 |

每条问题归档时记录：复现路径 / 影响 case 数 / 严重等级 / 修复建议。

## 时间线

| 阶段 | 天 | 产出 |
| --- | ---: | --- |
| 准备 | 1 | 沙盒克隆、9 case prompt 定稿、subagent 提示模板 |
| 试跑 | 1 | 跑 1 个 case 调通流程 |
| 全量 | 2 | 跑完 9 case |
| 验收 + 报告 | 1-3 | 打分、归档问题、写最终报告 |

## 通过门槛

- 单 case：≥ 80 分 + 无 P0/P1
- 整批：平均 ≥ 85 + 0 P0
- 任一 P0 → 整批不通过，触发 skill 维护流程

## 5 个推荐默认（已和用户确认）

1. 项目数 3 个：成本可控，复用基线
2. 执行深度到 Phase 5：才能验安全闸
3. 通过门槛 80/85：和现有 `VALIDATION` 对齐
4. skill 归属：RuoYi 双跑 1 个对照 + 其它 strict
5. 基线对比：和 `validation-runs/` 49 case 对比

## 与 `validation-runs/` 49 case 关系

| 维度 | 49 case | 本次 eval |
| --- | --- | --- |
| 项目 | RuoYi / Go / FastAPI / 等 | 复用其中 3 个 |
| prompt 风格 | 产品语言、pinned commit | 同 |
| 执行者 | 人写人审 | subagent 独立执行 |

| 产物 | 一份 md 报告 | docs + diff + 执行记录 |

## 风险与回滚

| 风险 | 缓解 |
| --- | --- |
| subagent 不调 skill | system prompt 强制 + 行为分激励 |
| 沙盒克隆失败 | 备选：worktree 或本地副本 |
| 9 case 跑不完 | 停止条件：连续 3 case < 70 即停 |
| 评分主观 | 机器分 + 人工分分轨，rubric 文字化 |
| Windows PowerShell 与 skill 默认 Bash 冲突 | 沙盒用 PowerShell 兼容路径 |
| subagent 平台不支持 fork | 退化为单会话内分阶段跑，行为记录手工 |
