# blue-xhs-knowledge

小红书知识卡片生成器。传入1个概念词语，智能规划叙事结构，生成3-6张轮播卡片图 + 小红书发布文案。

## 快速开始

```
/知识卡片 锚定
```

Claude 分析概念复杂度，规划叙事结构和视觉风格，生成轮播卡片组 + 小红书文案。

也可自然语言触发："帮我做一个锚定效应的知识卡片"、"生成锚定的图文帖子"、"概念可视化 心流"

## 四种风格

| 风格 | 模板 | 核心特征 | 源自 |
|------|------|----------|------|
| 空灵 | prompt-ethereal.md | 无地面，元素自由漂浮，大留白，诗意构图 | float (悬浮) |
| 实证 | prompt-grounded.md | 有承载面，元素落地生根，信息有"着陆面" | stage (舞台) |
| 语义 | prompt-semantic.md | 多层次叠加，语义链可视化，结构化递进 | deep (深度) |
| 强调 | prompt-emphasis.md | 同语义版 + 关键论点加粗/高亮，信息层级对比强烈 | deep-bold (深度强调) |

### 风格与叙事角色的搭配

叙事角色（封面/定义/场景/深度/结论）和视觉风格是独立的两层——Claude 根据概念气质将二者匹配：

- **封面/结论** → 倾向空灵（悬念引子、意象收束）
- **定义/场景** → 倾向实证（概念落地、生活实例）
- **深度解读** → 倾向语义或强调（多维展开、核心论点）

铁律：同一轮播组内，视觉风格尽量不重复（卡片数 > 4 时允许复用，但相邻卡片避免同风格）。

## 工作流程

| 步骤 | 内容 |
|------|------|
| 1. 存档检查 | 读取 concept-archive.md，拒绝已用概念 |
| 2. 叙事规划 | 分析概念复杂度，决定卡片数(3-6)、叙事角色、视觉风格分配 |
| 3. 生成 prompt | 按模板生成各卡片 prompt 文件，写入 batch 目录 |
| 4. 调脚本生图 | 运行 xhs-knowledge-generate.py，逐卡生成 PNG |
| 5. 追加存档 | 将本次概念写入 concept-archive.md |
| 6. 生成文案 | 输出小红书发布文案到 xhs-copy.md |

## 概念存档

每次生成的概念会自动存档到 `output/concept-archive.md`。
下次输入时，Claude 会检查存档，拒绝已用过的概念。

如果想重复某个概念，可以手动编辑 concept-archive.md 删除对应行。

## 与 blue-word-poster 的关系

| 维度 | 共同点 | 不同点 |
|------|--------|--------|
| API Keys | 复用 `D:\PyProject\blue-skillhub\.env` | — |
| 生图 Provider | GRS / Yunwu，模型 GPT-image-2 | — |
| 模板体系 | 4种风格，prompt 正文不可修改 | 模板文件不同（空灵/实证/语义/强调 vs 悬浮/舞台/深度/深度强调） |
| 生图脚本 | 各有独立脚本 | `xhs-knowledge-generate.py` vs `word-poster-generate.py` |
| 存档机制 | 词语/概念防重复 | concept-archive vs word-archive |
| 输出格式 | batch 目录 + prompt txt + PNG | 3-6张轮播 + 小红书文案 vs 4张海报 + 抖音文案 |
| 图片比例 | — | 3:4（小红书轮播） vs 9:16（竖版海报） |

## 红线

1. **prompt 正文不可修改** — references/ 中的模板是原文，只能填入 {{...}} 占位符
2. **图片比例固定** — 按 .env 配置（默认 3:4）
3. **必须输入 1 个概念词语** — 每次生成1组轮播
4. **概念不可与存档重复** — 防止重复使用
5. **同一轮播组内，视觉风格尽量不重复** — 卡片数 > 4 时允许复用，但相邻卡片避免同风格
6. **文案格式为 md** — 不是 org
7. **文案禁止 AI 腔** — 赋能/无缝/释放/震撼心灵等词汇禁止；禁止金句感；禁止引用图片
