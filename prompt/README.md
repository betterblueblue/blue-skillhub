# Prompt 工具箱

这里收录可以直接发给 AI 编码助手的实用 Prompt。它们适合解决具体协作问题，不要求安装 Skill，也不依赖某个客户端或插件。

每份 Prompt 都遵循三个原则：

- 不依赖特定客户端或插件。
- 说清楚需要提供什么、最终产出什么、怎样才算完成。
- 对容易失真的内容要求给出文件、命令或工作区证据。

## Prompt 列表

| Prompt | 什么时候拿出来用 | 你会得到什么 |
|--------|------------------|--------------|
| [开工前调研开源项目](open-source-project-research.md) | 已经有项目想法，但不想让 AI 凭空决定架构和技术栈 | 几个真正可比的开源项目，以及一份有依据、等你确认的实现方案 |
| [跨会话交接](session-handoff.md) | 今天先做到这里，明天或换个会话接着干 | 一份让新会话不用重新追问背景的 `HANDOFF.md` |
| [无交接恢复现场](workspace-recovery.md) | 会话突然中断，工作区只剩一堆没解释的改动 | 从 Git 和项目文件还原出的进度、卡点和未知项 |
| [卡住时重新梳理](stuck-reassessment.md) | 同一个报错反复出现，继续尝试也没有新信息 | 已确认事实、仍在猜的原因、已排除项和唯一下一步 |
| [找外援排障](expert-escalation.md) | 当前模型绕了多轮，还是找不到根因 | 外援模型拿到就能复现并接手的完整求援指令 |
| [需求变更对账](requirement-change-reconciliation.md) | 需求来回改，新旧要求快混在一起 | 唯一有效的当前需求、失效要求和返工影响 |
| [提交前整理改动](pre-commit-cleanup.md) | 准备提交，但正式改动和调试残留混在 diff 里 | 哪些该提交、哪些该清理、哪些绝对不能动的精确清单 |
| [生成独立验收指令](independent-review-request.md) | 实现会话说完成了，想换个模型从头核实 | 一份不采信原会话结论的独立验收指令 |

## 开工前调研开源项目怎么用

打开 [开工前调研开源项目](open-source-project-research.md)，把代码块里的 `[项目想法或 INTENT.md 路径]` 换成自己的内容，再把整个代码块发给能够访问 GitHub 的 AI。

如果你还说不清项目主要给谁用、解决什么问题，先用 IntentAnchor 生成 `INTENT.md`，然后把文件路径交给这个 Prompt。目标已经明确时，直接写一两段项目想法就够了。

AI 会先给出开源项目对比和推荐方案，不会直接写代码。你需要重点检查三件事：入选项目是否真的相似、结论有没有仓库证据、推荐方案有没有回到你的目标。确认方案后，再明确告诉它开始开发。
