# 卡住时重新梳理 Prompt

适用于同一个错误连续出现两次，或者连续尝试后没有获得新信息的情况。第一次正常的编译或测试失败不需要使用它。

```text
我们已经连续尝试，但同一个问题仍未解决。现在暂停修改文件，先重新梳理，不要继续碰运气。

请根据当前工作区和实际命令结果完成以下检查：

1. 重述当前目标和成功标准。
2. 写出第一次失败的完整命令、退出码和首个错误。不要只引用最后一次失败。
3. 按顺序列出已经尝试过的办法：改了什么、结果是什么、获得了什么新信息。
4. 分开列出已经有证据支持的事实和仍待验证的判断。
5. 说明当前为什么卡住：是在修改错误的位置、误解了报错、验证方式有问题，还是缺少必要信息。
6. 选择下一项最小验证动作。它必须能排除一个明确判断，不能同时修改多个地方。

梳理期间不要修改源码、配置或文档，不要安装依赖，不要还原现有改动，也不要创建临时调试文件。除非输入或前置条件已经变化，否则不要原样重跑同一条命令。

完成梳理后，只执行一个不会修改项目文件的最小验证动作，然后汇报结果。如果下一步必须写文件、改变外部数据或扩大任务范围，先说明原因和具体影响，不要直接执行。

输出必须包含：当前判断、关键证据、已经排除的原因、仍未确认的内容、下一步唯一动作。找不到证据时直接写“未确认”，不要猜。
```
